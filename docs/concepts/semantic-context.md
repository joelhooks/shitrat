# Semantic Context

ShitRat treats familiar configuration as a semantic document graph, not a markdown blob.

Prompt modules can be Markdown, MDX, or SVX. SVX is the example space because it plays well with pi-notes, but the durable contract is compiled harness prompt text.

Components such as `Identity`, `Rule`, `ToolPolicy`, `BrainPolicy`, `PrivateOverlay`, and `Receipt` are conceptual primitives. They do not need to render to UI to be useful. They let ShitRat validate, compose, parity-check, and explain context before installing it into Codex Desktop, Pi, Claude, or another harness.

Public defaults should be useful and opinionated. Private overlays should carry personal facts, machine config, secrets references, paid/private corpus references, and private skills.
